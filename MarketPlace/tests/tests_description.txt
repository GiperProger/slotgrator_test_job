Покрытие критических участков бизнес-логики: Проверка корректности работы классов Product, ProductImageManager и FavoriteManager.
Обратная совместимость: Убедиться, что клиенты, зависящие от поля image_url, продолжают получать ожидаемые данные, даже если добавлен новый массив images.
Валидация авторизации и обработки ошибок: Тесты для проверки корректного поведения API при отсутствии токена или при ошибках в запросе.
Cценарии для нового функционала: Комбинированное тестирование API эндпоинтов для избранного и списка товаров с реальными или замоканными зависимостями, имитирующими работу с AWS S3 и файловой системой.
